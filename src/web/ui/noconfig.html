Getting Started

Setup required

Symlinkarr needs a config.yaml before it can start. Choose an option below to get set up.

Wiki Getting Started Learn more User Guide

Option 1

Restore from backup

Place a Symlinkarr backup in your backup directory and restart. Config, database, and secrets are restored automatically.

Docker: mount the backup file into your backup directory and restart the container.

Local: run in your terminal:

symlinkarr restore <path-to-backup.json>

Use --dry-run to preview, --list to inspect backup contents before restoring.

Option 2

Create new installation

Generate a starter config.yaml and required directories for a fresh install.

Run in your terminal:

symlinkarr bootstrap

Use --list to check what is missing without creating anything. Docker users can skip this — compose already mounts config and data.

What a backup can restore: config.yaml, SQLite database, tracked symlinks, and config-local secretfiles.
Environment-only secrets and other external secret paths still need to be added manually.
Auto-restore: if a backup exists in the default backup directory when config.yaml is missing, Symlinkarr restores it automatically on startup. No manual step needed.
