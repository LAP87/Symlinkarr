# Symlinkarr Wiki Home

Symlinkarr keeps a Real-Debrid-backed media library clean and recoverable without hand-sorting real files.

This wiki is organized by what you are trying to do, not by code internals.

## Start Here

- Daily dashboard and triage: [Dashboard and Daily Operations](Dashboard-and-Daily-Operations.md)
- Run scans and understand skipped items: [Scan History and Skip Reasons](Scan-History-and-Why-Not-Signals.md)
- Fix broken paths safely: [Repair and Dead Links](Repair-and-Dead-Links.md)
- Review and apply cleanup work: [Cleanup, Audit, and Prune Preview](Cleanup-Audit-and-Prune-Preview.md)
- Create backups or recover a fresh install: [Backup and Restore](Backup-and-Restore.md)
- Validate setup and runtime safety: [Configuration and Doctor](Configuration-and-Doctor.md)
- Preview placements and inspect queued acquisition work: [Discover and Queue](Discover-and-Queue.md)
- Untangle legacy anime layouts: [Anime Cleanup](Anime-Remediation.md)
- Understand refresh behavior in Plex, Emby, and Jellyfin: [Media Server Refresh](Media-Server-Refresh-and-Deferred-Work.md)

## Common Flows

- First install or missing config: start with [Backup and Restore](Backup-and-Restore.md) and [Configuration and Doctor](Configuration-and-Doctor.md)
- Daily operation: start with [Dashboard and Daily Operations](Dashboard-and-Daily-Operations.md)
- Something failed during a scan: go to [Scan History and Skip Reasons](Scan-History-and-Why-Not-Signals.md)
- Media paths are broken: go to [Repair and Dead Links](Repair-and-Dead-Links.md)
- You want to remove stale or legacy junk: go to [Cleanup, Audit, and Prune Preview](Cleanup-Audit-and-Prune-Preview.md)
- Anime library is still split or blocked: go to [Anime Cleanup](Anime-Remediation.md)

## Scope

Use the wiki for:

- common tasks
- safety rules
- page-by-page web UI meaning
- media-server behavior
- install and recovery flows

Do not use the wiki as a changelog or engineering notebook. Those live in repo docs and `docs/dev-notes/`.
