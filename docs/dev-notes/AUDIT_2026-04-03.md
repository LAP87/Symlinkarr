## Summary Verdict

**Historical verdict only. Do not treat this as the current release gate.**

The project has a solid architectural foundation but has **19 CRITICAL issues** spread across all 6 audit areas. The issues are a mix of:
- **Data corruption risks** (repair rollback leaving orphaned symlinks, non-UTF-8 path truncation)
- **Security regressions** (auth bypass by default, CSRF on all mutating endpoints, broken Dependabot)
- **Functional bugs** (anime colon-stripping destroying titles, multi-episode files losing links, wrong API endpoints)

## RC Blockers (must fix before v1.0 RC)

### WEB AUTH BYPASS — Entire UI/API open by default
**File:** `src/web/mod.rs:1173-1207`

When no credentials are configured (the default), `guard_web_auth` short-circuits on every route because `!require_basic || basic_ok` evaluates to `true` when `require_basic` is `false`. Every UI page and API endpoint is completely open.

```rust
let authorized = if is_api {
    (!require_api_auth) || basic_ok || api_key_ok
} else {
    !require_basic || basic_ok   // <-- !require_basic is true when no basic auth configured
};
```

Impact: With `allow_remote=true` and no credentials (the default Docker compose setup), the entire UI and all API endpoints are unauthenticated. An attacker on the same network can read all library paths, symlink records, scan history, trigger destructive operations (scan, cleanup, prune, backup restore).

**Fix:** Require auth when `web.enabled=true` regardless of credential configuration. Fail startup (not warn) when `allow_remote=true && !auth_enabled()`.

### CSRF — No anti-CSRF token on any form
**File:** `src/web/mod.rs:1210-1258`

`guard_browser_mutations` only triggers when `method_requires_same_origin(method)` is true AND the request has browser metadata (Origin/Referer header). Without these headers, the entire guard skips. A victim visiting an attacker's page can have their browser silently trigger `POST /scan/trigger`, `POST /cleanup/audit`, `POST /links/repair`.

Additionally, the session token generator falls back to `nanos + PID` (predictable) if `getrandom` fails.

**Fix:** Add synchronizer token pattern. Include a hidden `<input type="hidden" name="csrf_token" value="{{ session_token }}">` in every form and verify it in `guard_browser_mutations`. Fail session token generation if all entropy sources fail.

### REPAIR DB/FS NOT CO-TRANSACTIONAL — Orphaned symlinks with no DB record
**File:** `src/repair.rs:958-1001`, `src/repair.rs:1219-1266`

Unlike `linker.rs` which wraps both DB insert and symlink creation in a single SQLite transaction, the repair path calls `repair_link` (filesystem-only) first, then separately calls `db.insert_link`. If the process crashes between the two, a symlink exists on disk with no DB record. Subsequent runs will not detect it as a fresh dead link (since the symlink is now correct), silently ignoring the orphaned original.

**Fix:** Wrap filesystem op and DB insert in a single transaction in `repair_all`.

---

### REPAIR ROLLBACK LEAVES DANGLING SYMLINK
**File:** `src/repair.rs:1003-1039`, `src/repair.rs:1232-1264`

When `db.insert_link` fails after a successful repair, `rollback_repair_link` tries to restore the original dead symlink. If the original source is still missing, the rollback refuses — but the error is caught and silently discarded. Result: a dangling symlink with no DB record, completely invisible to all subsequent operations.

**Fix:** On rollback failure, propagate the error rather than swallowing it. Alternatively, if the original source is gone, accept the new symlink state rather than leaving a dangling symlink with no DB record.

### NON-IDEMPOTENT REQUESTS BYPASS ALL RETRY PROTECTION
**File:** `src/api/http.rs:109-114`

```rust
pub async fn send_with_retry(request: RequestBuilder) -> Result<Response> {
    if request.try_clone().is_none() {
        return Ok(request.send().await?); // <-- single attempt, no retries, no timeouts
    }
```

When `try_clone()` returns `None` (multipart streams, streaming bodies), the request is sent once with no retry protection, no backoff, and no timeout enforcement beyond the global client timeout. Affects `DecypharrClient::add_content()` which uses `reqwest::multipart::Form`.

**Fix:** Buffer non-cloneable bodies into `Vec<u8>` before the retry loop.

---

### 401 NEVER RETRIED — No token refresh for most API clients
**File:** `src/api/http.rs:185-187`

```rust
fn is_retryable_status(status: StatusCode) -> bool {
    status == StatusCode::TOO_MANY_REQUESTS || status.is_server_error()
}
```

401 is not a server error, so it is never retried. TVDB manually handles 401 re-auth (only for series metadata — not episode fetching). Real-Debrid, Plex, Emby, Jellyfin, Prowlarr, Bazarr, Tautulli, Sonarr, Radarr: 401 is fatal with no recovery. Emby/Jellyfin tokens can expire. Real-Debrid can get spurious 401s at the transport layer.

**Fix:** Add 401 recovery to `send_with_retry` or a wrapper. For services with token-based auth, expose a re-auth callback.

### HTTP CLIENT BUILDER SILENTLY DEGRADES TIMEOUTS
**File:** `src/api/http.rs:98-107`

```rust
pub fn build_client() -> Client {
    reqwest::Client::builder()
        .connect_timeout(Duration::from_secs(5))
        .timeout(Duration::from_secs(20))
        .build()
        .unwrap_or_else(|_| Client::new()) // <-- silently ignores all errors
}
```

`Client::new()` uses reqwest's default 30-second timeout — not the configured 5/20 second limits. Any builder error (bad TLS config, bad DNS) silently degrades to 30-second timeouts, causing long hangs instead of fast failures.

**Fix:** Log a warning on builder failure, or `expect()` in debug/unwrap in release so failures are loud.

---

### HEALTH CHECKS ACCEPT HTML AS SUCCESS
**File:** `src/api/http.rs:177-181`

`check_system_status` returns `Ok(())` for any 200 OK, even if the body is HTML (proxy error page, redirect). Callers return `Result<()>` so the JSON parsing result is discarded. A 200 OK with an HTML body is treated as a successful health check.

**Fix:** Parse and validate the JSON in `check_system_status`, or return `Result<serde_json::Value>`.

---

### VACUUM RUNS SYNCHRONOUSLY, BLOCKING THE DB WRITER
**File:** `src/db.rs:2278` (housekeeping fn)

```rust
sqlx::query("PRAGMA optimize").execute(&self.pool).await?;
sqlx::query("VACUUM").execute(&self.pool).await?;
```

`VACUUM` can take seconds to minutes on large DBs and blocks all concurrent writers. With a 5-connection pool, this can exhaust the pool and stall all concurrent work.

**Fix:** Run `VACUUM` via `tokio::task::spawn_blocking` on a dedicated thread, or on a separate single-connection pool.

---

### NON-UTF-8 PATHS SILENTLY TRUNCATED TO EMPTY STRINGS
**File:** `src/db.rs:2225-2226`

```rust
let source_path = record.source_path.to_str().unwrap_or_default();
let target_path = record.target_path.to_str().unwrap_or_default();
```

Paths with valid but non-UTF-8 bytes silently become empty strings. Empty `source_path` can cause duplicate link records; empty `target_path` violates UNIQUE but produces a confusing error. The non-transactional `insert_link` correctly uses `path_to_db_text()` which returns an error instead.

**Fix:** Replace both `unwrap_or_default()` calls with `path_to_db_text()`.

### ANIME TITLES WITH COLONS LOSE ALL SUBTITLE WORDS
**File:** `src/utils.rs:171-186` (`normalize` function)

```rust
pub fn normalize(s: &str) -> String {
    s.to_lowercase()
        .chars()
        .map(|c| {
            if c.is_alphanumeric() || c.is_whitespace() { c } else { ' ' }
        })
        .collect::<String>()
        .split_whitespace()
        .collect::<Vec<_>>()
        .join(" ")
}
```

Colons become spaces, then `split_whitespace()` collapses consecutive spaces. `"Sword Art Online: Alicization"` → `"sword art online alicia"`. The word `"alicization"` is lost entirely. This affects every anime title containing a colon (extremely common: `"Attack on Titan: Final Season"`, `"Cowboy Bebop: A Cowboy's Work"`, `"Gintama: The Final"`).

**Fix:** Strip subtitle after `: ` or ` - ` before normalizing:
```rust
let s = s.split(": ").next().unwrap_or(&s).split(" - ").next().unwrap_or(&s);
```

### ANIME ABSOLUTE EPISODE QUERY USES ANIDB NUMBER (WRONG) FOR SCENE NUMBERING
**File:** `src/anime_scanner.rs:267-282`

When falling back to absolute-episode search, the code uses the AniDB absolute episode number (already offset-adjusted) as the scene/TVDB absolute episode. For shows with `episode_offset > 0`, this produces wrong Sonarr search queries (e.g., searching for episode 26 when scene numbering says episode 14).

**Fix:** Use `record.episode_number` (scene/TVDB episode) instead of the AniDB absolute episode for the fallback query.

---

### DOCKERFILE COPIES STATIC FILES TO WRONG LOCATION
**File:** `Dockerfile:22`

```dockerfile
COPY --from=builder /app/src/web/static /usr/local/bin/static
```

Static assets (`style.css`, `themes/`) are copied to `/usr/local/bin/static` — a binary directory. This happens to work because `static_dir()` looks in `<exe_dir>/static`, but it's fragile and misuses `bin` for data files.

**Fix:** Change destination to `/usr/local/share/symlinkarr/static` and update `static_dir()` fallback.

### DEPENDABOT HAS EMPTY PACKAGE-ECOSYSTEM — COMPLETELY NON-FUNCTIONAL
**File:** `.github/dependabot.yml:8`

```yaml
- package-ecosystem: "" # See documentation for possible values
```

Empty `package-ecosystem` means Dependabot will never open PRs for any dependency. Security patches for 35+ dependencies are never automated.

**Fix:**
```yaml
- package-ecosystem: "cargo"
  directory: "/"
  schedule:
    interval: "weekly"
```

### NO TEST COVERAGE INSTRUMENTATION ANYWHERE IN CI
**Finding:** No `cargo-llvm-cov`, no `tarpaulin`, no coverage reports. With ~50k LOC and no coverage gate, critical paths (linker, matcher, auto_acquire, repair) have zero measurable regression protection.

**Fix:** Add `cargo-llvm-cov` to CI with a minimum threshold (e.g., 60%) and fail the build if it drops.

---

### BINARY IS 24 MB, NOT STRIPPED, NO LTO
**Finding:** `target/release/symlinkarr` is 24 MB. `Cargo.toml` uses the default `release` profile with no `lto`, `strip`, or `opt-level` overrides. Comparable projects with `lto = "thin"` and `strip = true` are 8-12 MB.

**Fix:** Add to `[profile.release]`:
```toml
lto = "thin"
strip = true
opt-level = 3
panic = "abort"
```

### NO LICENSE HEADERS IN ANY SOURCE FILE
**Finding:** Zero of 46 `.rs` files contain `SPDX-License-Identifier`. The AGPL-3.0 license file exists at the repo root, but per-file licensing is ambiguous for contributors and users.

**Fix:** Add to every `.rs` file:
```rust
// SPDX-License-Identifier: AGPL-3.0-or-later

```

## HIGH Priority Findings

### `ProwlarrClient::grab` POSTs to wrong endpoint
**File:** `src/api/prowlarr.rs:116-143`

Posts to `/api/v1/search`. Correct endpoint is `/api/v1/release/{guid}`. Functional bug — grab may silently fail or have unexpected behavior.

---

### DNS failures not classified as retryable
**File:** `src/api/http.rs:189-191`

`is_retryable_error` does not include `err.is_dns_resolution()`. DNS glitches could exhaust retries without being classified as retryable.

**Fix:** Add `err.is_dns_resolution()` to the condition.

---

### Anime cutoff-upgrade logic is inverted
**File:** `src/anime_scanner.rs:172-178`

`CutoffUpgrade` skips when `!has_active` (no link). Should skip when `has_active` (already linked). Currently upgrades files with no link and skips files that need upgrading.

---

### Multi-episode files (`S01E01-E03`) silently lose links
**File:** `src/matcher.rs:802-811`

`destination_key` uses only `episode`, discarding `episode_end`. Two files with `episode=1, episode_end=3` both map to `DestinationKey::Tv { season: 1, episode: 1 }`. The reduce step arbitrarily keeps one and silently drops the rest.

---

### Jellyfin has no documented targeted-refresh API
**File:** `src/media_servers/jellyfin.rs:125-138`

Uses Emby-compatible `/Library/Media/Updated` which Jellyfin does not officially support for targeted library refresh. Users relying on targeted refresh may see stale library data.

---

### `insert_link` does not update `source_path` on conflict
**File:** `src/db.rs` — `insert_link` and `insert_link_in_tx` ON CONFLICT clauses

On `ON CONFLICT(target_path) DO UPDATE`, `source_path` is not in the UPDATE SET. If a source file moves, the old `source_path` persists in the DB even though the symlink on disk was updated.

---

### Secret URL+key consistency check gated behind `require_secret_provider`
**File:** `src/config.rs:991`

`validate_runtime_settings` only fires secret validation when `security.require_secret_provider` is `true`. A Prowlarr URL with an empty API key is accepted without error when that flag is `false`.

---

### `build_client()` creates a new Client per client instance
**File:** `src/api/http.rs:98-107`

Each of SonarrClient, RadarrClient, PlexClient etc. calls `http::build_client()`, creating many independent connection pools. Wasteful of file descriptors and TCP connections.

**Fix:** Use a shared `Client` via `static OnceLock<Client>`.

---

### Emby/Jellyfin have no token expiry handling
**File:** `src/media_servers/emby.rs:123-142`, `src/media_servers/jellyfin.rs:120-139`

If a refresh request returns 401, `send_with_retry` will not retry it, and the operation fails permanently without attempting re-authentication.

---

### `tokio = { features = ["full"] }` inflates compile time and binary size
**File:** `Cargo.toml:9`

`full` enables every Tokio feature. Only a fraction are used.

**Fix:** Replace with only the features actually used: `rt-multi-thread`, `sync`, `time`, `fs`, `io-util`.

## LOW / Nice-to-have

- `RateLimiter` global state never cleaned up (fine for daemon, awkward in tests)
- `DmmClient` uses non-standard custom hash (known trade-off for API compatibility)
- `AnimeIdentityGraph::load` returns `None` on anime-lists fetch failure (should be `Result`)
- `best_alias_score` tiebreaker favors shorter aliases (may disadvantage long titles)
- `absolute_episode > 50` heuristic for long-running anime is arbitrary
- No OVA/SP special episode variant handling in anime query builder
- `escape_sql_like_pattern` only escapes three characters (sufficient but worth documenting)
- Missing `Strict-Transport-Security` header
- No `robots.txt` handler
- No benchmark tracking in CI

## Positive Observations

The codebase demonstrates strong fundamentals in several areas:

- **Rate limiter design** — `RateLimiter` with `OnceLock` is solid for TMDB, TVDB, RD, and DMM
- **Retry backoff** — correctly exponential with jitter via bit-shift; `Retry-After` header parsing is proper
- **Linker atomic swap** — well-implemented with staging-path rename and rollback
- **Path traversal guards** — solid on report and backup paths
- **Session cookie** — `SameSite=Strict; HttpOnly` correctly configured
- **Credential comparisons** — `constant_time_eq` used for auth (with one noted exception)
- **Template escaping** — Askama's auto-escaping protects all `{{ }}` variables
- **DB schema** — parameterized SQL throughout, WAL mode, transactional migrations
- **Secret hygiene** — `security.require_secret_provider` mandates `env:` or `secretfile:` indirection
- **Anime TVDB re-auth** — correctly handles 401 with re-auth/retry loop
- **Bazarr/Tautulli dual auth** — correctly handles header + query param fallback
